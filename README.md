# Инвертированный индекс с дельта- и гамма-кодированием Элиаса для извлечения текстовых документов

## Описание
Проект представляет собой реализацию индексной структуры на основе инвертированного индекса, оптимизированной для хранения и поиска в больших массивах текстовых данных.

В качестве метода сжатия инвертированных списков применяется комбинация дельта-кодирования и гамма-кодирования Элиаса. Система спроектирована для обработки упоминаний, собранных из социальных сетей, и включает модули предобработки текста, построения индекса, булева поиска, а также инструментальный комплекс для сравнительного анализа эффективности сжатия и производительности поисковых операций.

## Алгоритм
### Дельта-кодирование
Преобразование строго возрастающей последовательности идентификаторов документов `D = [d_1, d_2, ..., d_n]` в последовательность разностей `Δ = [δ_1, δ_2, ..., δ_n]`, где `δ_1 = d_1 + 1`, `δ_i = d_i - d_{i-1} + 1` для `i > 1`. Смещение на единицу необходимо для обеспечения положительности значений, требуемой гамма-кодом Элиаса.

### Гамма-кодирование Элиаса
Переменный префиксный код для целых чисел `x ≥ 1`. Алгоритм кодирования включает:
1. Вычисление `k = ⌊log₂(x)⌋`.
2. Запись унарной части: `k` нулей, завершаемых единицей.
3. Запись бинарной части: младшие `k` бит двоичного представления `x` без старшего бита.
Декодирование выполняется последовательным чтением битового потока до встречной единицы, после чего считываются `k` бит для восстановления исходного значения.

### Булев поиск
Поддержка логических операций `AND` и `OR`. Запрос интерпретируется в соответствии с правилами приоритета: пробел соответствует операции конъюнкции, символ `/` - дизъюнкции. Слияние списков реализуется алгоритмом двух указателей с гарантией сохранения сортировки и удаления дубликатов. В сжатом режиме распаковка выполняется лениво, только для терминов, фигурирующих в запросе.

## Benchmark
Оценка производительности проводится по следующим метрикам:
- **Время индексирования**: суммарная длительность построения индекса и сериализации на диск.
- **Объём хранимых данных**: размер файлов индекса в байтах; расчёт коэффициента сжатия `CR = S_uncompressed / S_compressed` и относительной экономии дискового пространства.
- **Латентность поиска**: медианное время и 95-й перцентиль (`p95`) обработки контрольного запроса на основе 50 независимых прогонов.

## Установка

1. Клонировать репозиторий
2. Установить зависимости

   ```bash
   uv sync
   ```

## Использование
- Построение индекса:
  ```bash
  uv run src/main.py build -i data/telegram_results.csv -o data/index -c
  ```
  
- Выполнение поиска:
  ```bash
  uv run src/main.py search -d data/index -q *search_query* -c -l 20
  ```
  
- Запуск полного бенчмарка
  ```bash
  uv run src/main.py benchmark -i data/telegram_results.csv -o results --query *search_query* 
  ```
